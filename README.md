# 📚 library_econ_pipeline

공공도서관 경제 도서 데이터 ETL 파이프라인 — 수집 · AI 분류 · 분석 · 대시보드 · 리포트

> 정보나루 인기대출 데이터에서 경제 도서를 수집하고, AI로 분류·분석하여
> **"시민의 경제적 고민을 독서로 연결하기 위한 데이터 기반"**을 구축하는 로컬 파이프라인입니다.

---

## 1. 프로젝트 목적

- **1차 (미니 프로젝트)**: 크롤링 → ETL → AI 분류 → 분석 → 대시보드 → 리포트로 이어지는
  데이터 파이프라인 전체 흐름을 직접 구현하고 경험
- **2차 (확장 계획)**: 2026 도서관 데이터 활용 공모전 —
  "경제 정보격차 해소: 시민의 금융 고민을 공공도서관 데이터로 학습 경로화" 서비스의 데이터 기반

## 2. 아키텍처

```
┌─────────────────────── extract.py ───────────────────────┐
│  정보나루 loanItemSrch API (dtl_kdc=32) → 경제도서 + 대출수  │
│  네이버 도서 API → 책소개                                   │
│  알라딘 API → 판매지수                                      │
│  정보나루 이달의키워드 크롤링 (BeautifulSoup) → 키워드+가중치 │
└──────────────────────────┬────────────────────────────────┘
                           ▼
                 SQLite (data/library.db)
          raw_books / mart_books / monthly_keywords
                           ▼
┌─────────────────────── transform.py ──────────────────────┐
│  Gemini 2.5 Flash:                                         │
│  연령대(child/adult) + 난이도(D1~D3) + 주제(10종) 분류       │
└──────────────────────────┬────────────────────────────────┘
                           ▼
        pipeline.py (APScheduler, 매시간 자동 실행)
                           ▼
   analysis.py (분석·성과지표) → app.py (Streamlit) → report.py (.md)
```

## 3. 실행 방법

```bash
# 1) 환경 준비
python3 -m venv venv
source venv/bin/activate
pip install requests beautifulsoup4 pandas streamlit apscheduler google-genai python-dotenv lxml

# 2) .env 설정 (프로젝트 루트)
#    DATA4LIBRARY_API_KEY=...
#    NAVER_CLIENT_ID=...
#    NAVER_CLIENT_SECRET=...
#    ALADIN_API_KEY=...
#    GEMINI_API_KEY=...

# 3) 실행 순서
python3 src/db_init.py      # DB 초기화
python3 src/extract.py      # 데이터 수집 (약 2분)
python3 src/transform.py    # AI 분류 (약 12분)
python3 src/analysis.py     # 분석 리포트 (콘솔)
streamlit run src/app.py    # 대시보드
python3 src/report.py       # 마크다운 리포트 생성

# 자동화 (수집→분류 매시간 반복)
python3 src/pipeline.py
```

## 5. 핵심 분석 결과

1. **주제 공백**: 노후준비(4)·거시경제(3)·부동산(1)·돈관리(1)·기업분석(1) — 5권 미만.
   이 고민을 가진 시민은 대출 상위권에서 독서 경로를 찾기 어려움.
2. **정보 격차**: 도서관 전체 키워드는 사회이슈(차별·평등·혐오) 중심인데
   경제 도서는 투자·재테크 편중 → 경제교양·거시경제 경로 부재.
3. **AI 분류 검증 통과**: 어린이 도서 분리 후, D1 평균대출(8,447) > D2(7,226) —
   "쉬운 책일수록 많이 빌린다" 가설 지지로 분류의 합리성을 간접 확인.

## 6. 설계 결정 기록 (시행착오 포함)

이 프로젝트는 1차 시도(`library_skill_tree`)를 폐기하고 재설계한 결과물입니다.

| # | 문제 발견 | 결정 |
|---|----------|------|
| 1 | 4트랙×5레벨 설계에서 L2에 89% 쏠림, 보정 3종 모두 실패 | 원인은 계산식이 아닌 **데이터 한계**(대출 상위권=대중서, LLM 점수 분산 부족)로 결론. 설계 폐기 |
| 2 | `srchBooks`가 kdc 파라미터 무시 | `loanItemSrch` + `dtl_kdc` 발견 → 경제도서+대출수 직접 수집 |
| 3 | 알라딘·교보·YES24·네이버 모두 봇 차단/JS 렌더링으로 목차 수집 불가 | 목차 포기. 크롤링 요건은 **정보나루 이달의키워드**로 충족 → 오히려 트렌드 비교 분석 확보 |
| 4 | AI가 "어린이 vs 성인"을 "난이도"로 혼동 (D1=어린이책) | `age_group` 분리 + 성인 기준 난이도 재정의 → 재분류 후 검증 가설이 기각→지지로 반전 |
| 5 | 공모전 5개년 분석: 게임화 2회·성장플랫폼(책씨앗) 기출 | 포지셔닝을 "경제 정보격차 해소"로 변경. 미니 프로젝트와 공모전 스코프 분리 |

## 7. 한계 및 개선 방향

- **데이터 소스 편향**: 대출 상위 200권은 인기 대중서 위주 → 심화서(D3) 3권뿐.
  개선: 신간 API·전체 장서 데이터로 소스 다변화
- **목차 미수집**: 분류 입력이 책소개(description)에 의존.
  개선: 국립중앙도서관 서지 API 재시도 또는 유료 협의
- **스케줄러의 실효성**: 대출 순위 데이터는 변동이 느려 매시간 실행의 의미가 제한적.
  신간·키워드처럼 자주 바뀌는 소스에 붙이면 가치 상승
- **AI 분류 검증**: 대출수 상관은 간접 검증. 전문가 레이블링(샘플 수동 검증) 추가 여지

## 8. 공모전 확장 계획 (2026-08)

- 고민 기반 진입: "돈이 안 모여요" / "경제 뉴스가 어려워요" → 목표(Goal) 매핑
- 목표 → 단계(개념 이해/실행 방법/사례 적용, 복수 태그) → 세부 태그 탐색
- 본 파이프라인의 분석 결과(주제 공백·정보 격차)가 서비스 필요성의 데이터 근거
